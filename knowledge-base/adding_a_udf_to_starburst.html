    Adding a UDF to Starburst Enterprise #

    Starburst Enterprise allows the addition of User Defined Functions (UDFs) written as Java packages. The framework Starburst Enterprise uses is distinct from Hive, but Hive UDFs can typically be transferred into this framework with a minimal amount of modification.

    Create the package #

    The easiest way to build a UDF for SEP is to get the pom.xml file from the matching version of the trino-teradata-functions directory.

    Once you have downloaded the pom.xml file, pick a name for your UDF package, create the directory in a path appropriate to your environment, and copy the downloaded pom.xml file into it. In this example, the package name is trino-stringfunctions:

    $ mkdir -p trino-stringfunctions/src/main/java/io/prestosql/stringfunctions/
$ cp ~/Downloads/pom.xml presto-stringfunctions

    Next, edit the file pom.xml file, making the following changes:

    • change the artifactId to your package name (In this example, the artifactId becomes trino-stringfunctions)
    • remove the io.airlift.concurrent dependency
    • remove the joda-time.joda-time dependency
    • remove the org.antlr.antlr4-runtime dependency

    The pom.xml file for your standalone UDF by default disables the git checkin plugin. Re-enable this to have Maven check your UDF into your own repository, and make the following change to allow the Maven violations plugin to work correctly:

    $ mkdir presto-stringfunctions/src/modernizer
$ aws s3 cp s3://starburstdata/support/example_udf/violations.xml presto-stringfunctions/src/modernizer

    Open a project in your IDE (we highly suggest using IntelliJ for this UDF and import the modified pom.xml file in to ensure that the annotations and types needed for this UDF to work in Starburst are correct. Because Starburst uses strict style checking, you will want to import Airlift’s Code Style project as well.

    Finally, add the Apache license to your project.

    Create the plugin class #

    A plugin class is required in all UDF packages. Starburst will call the class to list the functions available. This class should add all of the classes in which you have declared UDFs. You can simply chain the add() functions to add more.

    package io.prestosql.stringfunctions;

import com.google.common.collect.ImmutableSet;
import io.prestosql.spi.Plugin;

import java.util.Set;

public class StringFunctionsPlugin
        implements Plugin
{
    @Override
    public Set<Class<?>> getFunctions()
    {
        return ImmutableSet.<Class<?>>builder()
                .add(StringComparisonFunctions.class)
                .build();
    }
}

    Add a UDF class to the package #

    This section shows how to create an example class to compare strings, the StringComparisonFunctions class. To the stringfunctions directory we created above, add a file named StringComparisonFunctions.java consisting of the following:

    package io.prestosql.stringfunctions;

public final class StringComparisonFunctions
{
}

    Add a function to the class #

    Once the class is created, add three annotations to the function to specify the name, return type, and a description. Do this for all functions:

    @Description("Checks if two strings are anagrams of one another.")
@ScalarFunction("are_anagrams")
@SqlType(StandardTypes.BOOLEAN)

    Next, we will add a function signature with our inputs, including annotations specifying the SQL input types, and the session information:

    public static boolean are_anagrams(ConnectorSession session,
    @SqlType(StandardTypes.VARCHAR) Slice string1,
    @SqlType(StandardTypes.VARCHAR) Slice string2)

    Add the function logic, a constructor, and all the necessary imports, resulting in the following:

    package io.prestosql.stringfunctions;

import io.airlift.slice.Slice;
import io.prestosql.spi.connector.ConnectorSession;
import io.prestosql.spi.function.Description;
import io.prestosql.spi.function.ScalarFunction;
import io.prestosql.spi.function.SqlType;
import io.prestosql.spi.type.StandardTypes;

import java.util.Arrays;

public final class StringComparisonFunctions
{
   private StringComparisonFunctions()
   {
   }

   @Description("Checks if two strings are anagrams of one another.")
   @ScalarFunction("are_anagrams")
   @SqlType(StandardTypes.BOOLEAN)

   public static boolean are_anagrams(ConnectorSession session,
           @SqlType(StandardTypes.VARCHAR) Slice string1,
           @SqlType(StandardTypes.VARCHAR) Slice string2)
   {
       byte[] string1Sorted = string1.getBytes();
       Arrays.sort(string1Sorted);
       byte[] string2Sorted = string2.getBytes();
       Arrays.sort(string2Sorted);
       return Arrays.equals(string1Sorted, string2Sorted);
   }
}

    Including external dependencies #

    To include external dependencies, add the dependency to your pom.xml and import the package as you normally would. In this section, we will use the Levenshtein Distance function implementation from Apache Commons in this trino-stringfunctions package example. To use it in the package, add the following to the pom.xml file, under the dependencies sections:

      <!-- https://mvnrepository.com/artifact/org.apache.commons/commons-lang3 -->
       <dependency>
           <groupId>org.apache.commons</groupId>
           <artifactId>commons-lang3</artifactId>
           <version>3.7</version>
       </dependency>

    Add the following to the imports section in the existing StringComparisonFunctions class:

    import org.apache.commons.lang3.StringUtils;

    Finally, add an annotated function that uses the imported function to the StringComparisonFunctions class:

       @Description("Measure the Levenshtein distance between strings.")
   @ScalarFunction("lev_dist")
   @SqlType(StandardTypes.INTEGER)

   public static int lev_dist(ConnectorSession session,
                             @SqlType(StandardTypes.VARCHAR) Slice string1,
                             @SqlType(StandardTypes.VARCHAR) Slice string2)

   {

       return StringUtils.getLevenshteinDistance(string1.toStringUtf8(), string2.toStringUtf8());

   }

    Building the package #

    Compile your package into a set of JAR files to add to your plugins folder in Starburst by running the following command from your package directory:

    $ mvn clean package

    This creates a target directory containing a trino-stringfunctions-<presto_version> directory with your JAR files, as well as a .zip file with the same contents.

    Deploying the package to Starburst #

    Copy the new directory and its contents to the Presto plugin directory on every node, typically located in /usr/lib/starburst/plugin, and ensure the executable permissions are correct.

    $ chmod 755 *.jar

    You must restart the starburst coordinator and worker nodes to make your UDF should be visible and ready to use.

    Using your UDF #

    The functions defined in the example class, are_anagrams and lev_dist, are accessed as follows in Starburst Enterprise:

    starburst> select are_anagrams(‘listen’,’silent’); _col0 ——- true

    starburst> select lev_dist(‘listen’,’listerine’); _col0 ——- 3
