Trying SEP on any Linux

You can install Starburst Enterprise Presto (SEP) on any 64-bit Linux distribution with the following steps:

  1. Obtain the most recent SEP tar.gz archive
  2. Unpack the archive as root
  3. Add configuration files
  4. Start the SEP server
  5. Obtain the Presto CLI client and run tests

Prerequisites

SEP requires a Linux distribution that:

  • Is no more than a few years old
  • Runs on 64-bit Intel hardware
  • Has Java 11.0.7 or a later Java 11 release. Java distributions from Oracle, OpenJDK, or Azul are supported. (Newer Java releases may work but are not tested.)
  • Has Python 2.7 or later (only to run the launcher utility)

(SEP also installs for evaluation purposes only on a macOS release with the same prerequisites.)

Download an SEP archive

To gain access to SEP archives, visit the Starburst website and click the Download Now button.

This opens a dialog that prompts for your name, email address, and location. Fill out the form using a valid email address, then click Download Starburst free.

A few moments later, you receive email from Starburst with a link to the downloads page.

Note You can optionally reply to this email to request a trial license that unlocks the performance and security enhancements of SEP. However, to get a SEP server up and running quickly, you can postpone using a license for now.

The Downloads page is organized into Long-Term and Short-Term Support sections. Choose an LTS edition unless you know that a certain feature or fix requires a newer STS edition.

Download the tar.gz from the All Platforms LTS link. For example, download presto-server-345-e.1.tar.gz.

Download the Presto CLI Jar file

While you’re on the download page described above, download the Presto CLI client Jar file that matches the version of the server you downloaded. For example, download presto-cli-345-3.1-executable.jar. We will take advantage of this file later.

Unpack the archive

The contents of the tar.gz archive are by default owned by root. Select an appropriate location and unpack the files as usual. (This page uses /usr/local as the location.) For example:

cd /usr/local
sudo tar xvzf ~/Downloads/starburst-presto-server-345-e.1.tar.gz

This creates a directory named the same as the basename of the tar.gz file in the current directory. This is the presto-root directory.

If you do not want the SEP server to run as root, create a user named presto and change ownership of all files and directories in the new presto-root directory, or otherwise follow your site’s standards for running servers.

Inspect the new directory to find that it contains three directories:

bin
lib
plugin

Add configuration files

Even the simplest Presto server must have a minimum set of configuration files before it can start.

The simplest way to get started quickly is to use the set of configuration files provided as examples for the O’Reilly book Presto: The Definitive Guide. Download these samples from their GitHub location either as a zip file or a git clone.

From your unzip or clone location of these samples, copy the entire etc directory from the single-installation sample to your presto-root directory. For example:

cd <unzip-or-clone-of-Def-Guide-samples>/single-installation
sudo rsync -av etc /usr/local/presto-server-345-e.1/

Start the server

From presto-root, run the following command:

sudo bin/launcher start

The launcher script returns a process ID for a successful server start.

For a presto-root in which you changed file and subdirectory ownership, you can drop the sudo or su presto before running bin/launcher.

Use the Presto CLI to validate the server

Download the most recent Presto CLI Jar from the SEP download page as described in Download the Presto CLI Jar file above.

This is an executable Jar file that is usable as-is. To make it practical, copy it to a directory in the PATH and rename it. For example:

cd /usr/local/bin
cp -p /home/<yourname>/Download/presto-cli-345-e.1-executable.jar .
mv presto-cli-345-e.1-executable.jar presto
chmod +x presto

Now run presto to connect to the running server:

presto

This opens a Presto CLI shell, with presto> prompt. All commands entered here must be terminated with a semicolon.

Exit presto’s interactive mode with quit; or exit;:

presto> quit;

To get started with the presto command, see First steps with the Presto CLI.
