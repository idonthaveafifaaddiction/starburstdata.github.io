Trying out Starburst products

Starburst makes it easy to set up trial configurations of Starburst Enterprise Presto (SEP) and Starburst Galaxy.

Up and running fast

Use a local machine installation to get SEP running quickly as a demo or trial system for your site’s developers.

On any Linux distro, you can:

  • Set up SEP with the tar.gz installation.

  • Set up SEP as a cluster of one or more Docker containers.

Both installation methods can be installed without a SEP license, or to unlock the performance and security enhancements of SEP, you can request a license as described in the linked pages above.

For initial evaluation only, you can use either method on recent releases of macOS. MacOS is explicitly not supported to run Presto services for any other purpose.

Consider the following ways to install and evaluate Starburst Enterprise Presto and Starburst Galaxy, listed in ease of use order.

Starburst Galaxy: web-based management

Starburst Galaxy is a web-based interface that allows you to create and manage Presto clusters to access the data you need wherever it lives.

To begin, you need:

  • A running SEP server, which can be:
    • The URL for a locally running server with at least minimum configuration.
    • The credentials for SEP running on an Amazon AWS instance.
  • The URLs and access credentials for the data sources you want Starburst Galaxy to manage.

See Starburst Galaxy for the next steps.

SEP managed by a commercial cloud provider

The marketplace collections of international cloud providers include a SEP option. These let you purchase a preconfigured machine image to run on their cloud hosts under your control. A free trial configuration is available from most providers.

Cloud provider images of SEP are available from:

  • Microsoft Azure
  • Amazon Web Services
  • Google Cloud Platform
  • Red Hat Marketplace

For all marketplace implementations, you need:

  • Access credentials from your cloud provider.
  • The URLs and access credentials for the data sources you want your cloud-hosted SEP cluster to access.

SEP managed on a Kubernetes cluster

You can install SEP on a Kubernetes cluster, taking advantage of Kubernetes abstractions to free you from the details of servers, hosts, and IP addresses.

Starburst recommends using Helm charts to configure your Kubernetes cluster as described in Kubernetes with Helm. A direct Kubernetes operator is also available.

You can install SEP on a private Kubernetes configuration on a local network or on a cloud provider’s Kubernetes offering, including:

  • Microsoft Azure Kubernetes Service (AKS)
  • Amazon Elastic Kubernetes Service (EKS)
  • Google Kubernetes Engine (GKE)
  • Red Hat OpenShift (requires OpenShift 4.x or later)

As with the cloud provider marketplace offerings, you need:

  • Access credentials from your cloud provider.
  • The URLs and access credentials for the data sources you want your Kubernetes-hosted SEP cluster to access.

SEP with template-driven cloud configuration

You can configure a SEP cluster on Amazon AWS using a template that describes the AWS resources you need. The Amazon Cloud Formation Template can then create any number of AWS instances so defined.

SEP with full local control

You can install and configure a SEP cluster yourself on a local network or on a cloud installation that you control.

The advantage of this is that you manage and control all aspects of your SEP cluster.

The disadvantage is the same, and troubleshooting cluster problems requires knowledge of and access to your configuration settings.

You can configure a local SEP cluster in several ways:

  • On a local Kubernetes cluster using Helm charts.
  • On a local Kubernetes cluster using the SEP Kubernetes operator.
  • On a local network of individual machines without Kubernetes.

To install on local machines you have several options:

  • For RPM-based Linux distros, use the presto-admin tool.
  • For any Linux distro, set up SEP with the tar.gz installation.
  • For any Linux distro, set up SEP as a cluster of Docker containers.
