Defining, configuring and securing catalogs

Catalogs are a key concept in Presto, as they represent the data sources the cluster will access.

Catalog definition

In Presto, a catalog defines the schemas in a data source that are accessible through a connector. Catalogs are defined in Presto by configuration files. Each catalog you define will specify one and only one connector using the connector.name property. You must select the correct connector for your data source, such as Redshift, Hive or MySQL.

Note:
While a catalog can have only one connector, a single connector may be associated with multiple catalogs.

You can read about adding catalogs here.

Configuring catalogs

Depending on your data source and the connector it must use, a number of different configuration properties will be available. Connector-specific properties are defined in the documentation for each connector. A list of all connectors is available here.

Many connectors share common configuration properties with the base Hive connector as well. Look for a link to the Hive connector documentation in your connectors’ documentation. If it is there, you can use those additional configuration properties as indicated.

In some cases, Starburst Enterprise Presto (SEP) offers extended versions of open source Presto connectors. In these cases, any additional configuration properties that are exclusive to the extended SEP connector will be documented in the SEP documentation, and a link provided to the open source documentation with the default property set.

Securing catalogs

How your catalogs are secured depends on how your Presto cluster itself was secured, and whether your organization uses LDAP and access control tools such as Ranger. SEP can be configured to provide security at the catalog-level, but that security configuration is not recommended.

If you are data engineer managing data sources in an existing Presto cluster, check with your SEP administrator to learn how security is configured for your cluster.

If you are an administrator setting up a new cluster, and want to learn how to secure your data, you’ll want to read our guide on securing Presto.
