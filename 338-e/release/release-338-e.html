Release 338-e LTS (xx Jul 2020)#

Starburst Enterprise Presto 338-e is the follow up release to the 336-e STS release and the 332-e LTS release:

  • Highlights since 332-e
  • Breaking changes since 332-e
  • Significant Starburst improvements since 336-e

The 338-e release includes all improvements from the Presto project releases:

  • Presto 333
  • Presto 334
  • Presto 335
  • Presto 336
  • Presto 337
  • Presto 338

This release is a short term support (STS) release and contains all improvements from the Starburst Enterprise releases releases since 332-e LTS:

  • 334-e STS
  • 336-e STS

Highlights since 332-e#

  • MapR Hive metastore support
  • Hive connector storage caching
  • Native table operator for the Teradata connector
  • Support for version 7.x of the Cloudera Data Platform (CDP)
  • Fix for CVE-2020-15087xxx
  • Add Apache Druid connector
  • Add Apache Pinot connector
  • Add Prometheus connector
  • Improve authentication support for the Web UI
  • Elasticsearch DSL passthrough support in the Elasticsearch connector

Breaking changes since 332-e#

  • Java 11 is now the required runtime for Presto servers. More information can be found in in the Java requirements section.

Significant Starburst improvements since 336-e#

  • Improvements of the native table operator for the Teradata connector
  • Add support for caching for distributed file systems requiring Kerberos authentication
  • Update JDBC driver used by the Snowflake connector
  • First limited support for aggregation pushdown in the Oracle connector and the Teradata connector
  • Add support SQL-based management of Hive Ranger policies
  • Improve audit support for system level security with Ranger
  • Nanosecond support for timestamp with and without time zone
