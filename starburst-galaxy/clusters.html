    Clusters #

    A cluster provides the resources to run queries against numerous data sources. Clusters define the number of workers, the configuration for the JVM runtime, configured data sources, and others aspects.

    Starburst Galaxy allows you to create, edit, and delete clusters from the UI. Access your clusters at any time by clicking Clusters on the left hand menu.

    Cluster details include:

    • Name
    • Status
    • Action
    • Workers
    • Data sources
    • Start date

    Add a new cluster #

    Before you can create a cluster in Starburst Galaxy, you need to add one or more data sources.

    Basic information #

    Identify the cluster and select the data sources it accesses.

    1. From the Cluster page, select + New. (If this is your first cluster, click + New cluster)
    2. In Cluster name, enter a unique name for your cluster
    3. Click Add tags to select the key value pairs associated with your data source. Click Add tags to save your selections
      • Tags are optional, you can create a new tag to use from this page
      • Create a new tag: Click + Add another > select Key > enter Value > click Add tag
    4. Select one or more Data sources
    5. Click Next

    Configure Presto #

    Set the Presto configuration options for this cluster. (AWS instance type link)

    1. Select an instance type to use for the Presto coordinator and workers
    2. Enter the Minimum number of workers and Maximum number of workers
    3. Select Show more options to see the Presto coordinator and Presto workers
    4. Enter Bootstrap script URI
    5. Select the Starburst license file option
    6. Click Next

    Configure LDAP #

    You need to configure the connection to your LDAP server. This allows users from your LDAP system to authenticate with Starburst Galaxy for administration or analysis work.
