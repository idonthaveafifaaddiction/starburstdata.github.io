    LDAP authentication #

    The LDAP configuration enables users to authenticate to Starburst Galaxy.

    It also allows you to use the same credentials to connect with client tools, such as the Presto CLI, or an application using the JDBC or ODBC driver. Once connected you can use them to run queries against the data sources configured as catalogs.

    The following properties need to be configured for your LDAP authentications support. Work with the administrator of your LDAP server to determine the correct values.

    Property Description
    URL The combination of the protocol, FQDN and port used by your LDAP server. Protocol can be ldaps:// or ldap://
    User bind pattern The pattern used to look up the relevant users in your LDAP directory. An example is ${USER}@ldap.example.com
    Group authorization query Queries the LDAP user groups
    Presto distinguished name Starburst Galaxy service user identity
    Bind password Authorizes the user bind pattern
    User distinguished name Service user identity
    Cache TTL Time interval to cache credentials after first authentication, and before authentication is required again

    Consult the LDAP documentation for SEP for more details, and work with your LDAP server administrator.

    Edit LDAP cluster settings #

    Edit your cluster’s LDAP settings from the Clusters page.

    1. Locate the cluster whose LDAP settings you want to edit
    2. Select the Options icon (three stacked dots), click Edit
    3. Select the LDAP configuration tab
    4. In the Configure LDAP properties section, edit your properties
    5. When you complete your edits, click Update to save your new configuration
    6. To apply your changes, restart your cluster
