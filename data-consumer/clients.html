    Starburst clients #

    Starburst Enterprise platform (SEP) lets you query anything, anywhere. And, it lets you federate your favorite data sources into a single query without waiting on your data ops team to deliver complex ETL.

    To get started querying data through SEP, you’ll need the right client. Starburst provides both Java Database Connectivity (JDBC) and Open Database Connectivity (ODBC) drivers to connect to your SEP cluster. Which one you need depends on your data toolkit.

    Connection information #

    No matter how you connect to SEP, you’ll need some basic connection information to setup your driver and tools:

    • JDBC URL
    • ODBC Data Source Name (DSN) or connection string
    • Your credentials

    We recommend that you check in with your data ops team to get this information if you don’t already have it before proceeding to install one or more SEP clients.

    SEP JDBC driver #

    The JDBC standard provides for truly platform-independent connectivity and requires no license. It uses the Java programming language and translates between Java-language calls, and calls that your data source understands. The SEP JDBC driver is what most users use to connect their tools to SEP. You likely already use one with your current toolkit, but if you are unsure, check your tool’s documentation.

    Download the latest SEP JDBC client here.

    SEP ODBC driver #

    Unlike the JDBC driver, the SEP ODBC driver requires a license. Check with your data ops team to get the licensed binary. Information on ODBC driver installation and system requirements can be found here.

    SEP Power BI DirectQuery driver #

    If you use Microsoft Power BI Desktop and Microsoft Power BI Service, you must use the SEP Power BI DirectQuery driver on top of the ODBC driver to connect to SEP. You can read how to obtain and install the .mez driver here.

    Command Line Interface #

    SEP uses the SEP Command Line Interface (CLI) to provide an interactive shell for running queries in a terminal. You can also use the CLI to inspect SEP metadata. The CLI is a self-executing JAR file, which means it acts like a normal UNIX executable.

    You can download and learn how to use the CLI here.

    Tableau #

    As a Tableau user you will likely use the same type of driver, JDBC or ODBC, as you do now to connect Tableau to SEP, except you must use the SEP-specific driver. Read more about that here.

    Other resources #

    The O’Reilly book, Presto, the Definitive Guide was written here at Starburst, and contains some great information on getting started with using different types of clients with SEP. It’s available for free!
